技术路线探讨

  • AI
  • Prolog
  • language
Tech

在应用功能和商业模式确定下来之后，项目组中的开发人员对接下来的开发之路进行了较为详细的展望，并且提出了以下的技术路线;

  1. 首先是数据方面。数据主要来源于网页、贴吧以及微博上用户所发布的安全数据。例如，在何时何地发生了抢劫、偷窃等涉及人身安全的事件。网页和贴吧的数据主要依靠网络爬虫在区域内搜索关键字。网络爬虫是一个自动提取网页的程序，继而通过正则表达式匹配目标文本来得到目标数据。而微博上的数据则是来源于微博用户@项目组的官方微博账号以及微博上关键字的搜索，再将这些信息整合为有效数据。在得到数据之后就是对数据的分析了。在分析之前，项目组先将数据整合成统一的格式。例如，以时间/地点/事件的格式消除冗余信息。分析前期，依靠手动判断信息的有效性，并利用这些数据以机器学习的方式训练我们的人工智能，以正样本（当信息有效时）和负样本（当信息无效时）作为反馈。若机器判断的准确性足够高，则数据将完全由其判断。
    当数据的规模到达一定的程度后，就可以利用这些信息判断一个地区的安全程度。而对于该指标的评估也会随着时间的推移而变化，因为信息的可信度绝大部分取决于信息的时效性。
    项目的一个重要的目标是数据可视化数据可视化指是将数据库中每一个数据项作为单个图元元素表示，大量的数据集构成数据图像，同时将数据的各个属性值以多维数据的形式表示，可以从不同的维度观察数据，从而对数据进行更深入的观察和分析。在应用中，我们将根据地区的安全评级绘制图层覆盖在地图上，给用户方便直接的使用体验。
  2. 很关键的一个技术便是安卓客户端的开发。移动客户端主要在Intellij Idea上进行开发。拥有强大类库的Java语言又为安卓开发提供了方便的开发工具。在地图相关功能上，通过调用百度地图API（Application Programming Interface）实现，包括地图的显示、用户的定位、目的地的搜索、图层的覆盖等。
  3. 另一个不可或缺的部分便是数据库了。JDBC（Java Data Base Connectivity）是一种用于执行SQ语句的Java API，可以为多种关系数据库提供统一访问。JDBC为数据库开发人员提供了标准API，据此可以构建更高级的工具和接口。与数据库建立连接的标准方法是调用DriverManager.getConnection方法。该方法接受含有某个URL的字符串，尝试找到可以与该URL所代表的数据库进行连接的驱动程序。
    当然，随着用户量的增加和时间的推移，数据库中的数据会越来越多。项目决定采用分期、分种类、不同案件类型、不同时间进行数据存储。
  4. 还有一个技术是和服务器有关。服务器系统架构采用Java Platform Enterprise Edition的多层框架结构模型。服务器使用HTTP协议实现客户端和服务端的连接。服务器端的数据处理采用JSON（JavaScript Object Notation）这种轻量级的数据交换格式。JSON使用了类似C语言家族的习惯，使其成为理想的数据交换语言，易于阅读和编写的同时也易于及其解析和生成。
